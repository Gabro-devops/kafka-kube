# Jenkins CI/CD for Kafka — Complete Guide

## Overview

Jenkins is used to automate Kafka operations — creating topics, managing users, and monitoring the cluster. All pipelines run as dynamic Kubernetes agent pods.

---

## Pipelines

### 01-Kafka-Manage-Topic

Creates or deletes a Kafka topic via a KafkaTopic Strimzi resource.

**Parameters:**
| Parameter | Description | Default |
|---|---|---|
| ACTION | CREATE or DELETE | CREATE |
| TOPIC_NAME | Topic name | (required) |
| PARTITIONS | Number of partitions | 3 |
| REPLICAS | Replication factor | 3 |
| RETENTION_HOURS | Retention in hours | 168 (7 days) |
| CONFIRM_DELETE | For DELETE only: retype topic name | — |

### 02-Kafka-Manage-User

Creates or deletes a Kafka user (KafkaUser) with SCRAM-SHA-512 authentication and ACLs based on role.

**Parameters:**
| Parameter | Description |
|---|---|
| ACTION | CREATE or DELETE |
| USERNAME | Username |
| ROLE | producer / consumer / producer-consumer / admin |
| TOPIC_PATTERN | Topic pattern (e.g. `orders-*` or `*` for all) |
| CONSUMER_GROUP | Consumer group |
| CONFIRM_DELETE | For DELETE only: retype username |

**ACLs generated by role:**
| Role | ACLs |
|---|---|
| producer | Write, Describe, Create on topics |
| consumer | Read, Describe on topics + Read on consumer group |
| producer-consumer | Both of the above |
| admin | All on topics, groups, cluster |

### 03-Kafka-Cluster-Health

Checks the overall cluster status: pods, Kafka CRD, topics, users.

Runs automatically every 30 minutes via a schedule configured in AWX.

---

## Architecture

Jenkins runs as a Deployment in the `kafka-lab` namespace. Each pipeline build creates a temporary agent pod with two containers:
- `kubectl` — for Kubernetes operations
- `kafka-tools` — for direct broker operations (only in health check)

The agent uses the `jenkins-admin` ServiceAccount which has a ClusterRoleBinding with `cluster-admin` permissions on the namespace.

## Useful Commands

```bash
# Check Jenkins pod
kubectl get pods -n kafka-lab -l app=jenkins

# View logs
kubectl logs -l app=jenkins -n kafka-lab

# Restart Jenkins
kubectl rollout restart deployment/jenkins -n kafka-lab

# List topics created by Jenkins
kubectl get kafkatopic -n kafka-lab -l managed-by=jenkins
```

## Troubleshooting

**Pipeline fails with "permission denied":**
```bash
kubectl describe clusterrolebinding jenkins-admin-binding-kafka-lab
```

**Agent pod does not start:**
```bash
kubectl get pods -n kafka-lab -l jenkins=agent
```

**Pipeline cannot find the repo:**
Make sure the GitHub repo `https://github.com/Gabro-devops/kafka-kube.git` is public and accessible.
